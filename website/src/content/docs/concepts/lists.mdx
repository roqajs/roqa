The `<For>` component efficiently renders lists of items. It uses a sophisticated reconciliation algorithm to minimize DOM operations when items are added, removed, or reordered.

## Basic Usage

```jsx
import { defineComponent, cell, get } from 'rift-js';

function TodoList() {
  const todos = cell([
    { id: 1, text: 'Learn Rift' },
    { id: 2, text: 'Build an app' },
    { id: 3, text: 'Deploy to production' },
  ]);

  return (
    <ul>
      <For each={todos}>
        {(todo) => <li>{todo.text}</li>}
      </For>
    </ul>
  );
}

defineComponent('todo-list', TodoList);
```

<Aside>
`<For>` is a **compile-time** component. It's one of only two PascalCase JSX elements allowed in Rift (along with `<Show>`). It compiles to efficient `forBlock()` calls.
</Aside>

## The `each` Prop

Pass a cell containing an array to the `each` prop:

```jsx
const items = cell(['Apple', 'Banana', 'Cherry']);

<For each={items}>
  {(item) => <li>{item}</li>}
</For>
```

When the array changes, Rift efficiently updates only the affected DOM nodes.

## Accessing the Index

The render function receives the index as its second parameter:

```jsx
const fruits = cell(['Apple', 'Banana', 'Cherry']);

<For each={fruits}>
  {(fruit, index) => (
    <li>
      {index + 1}. {fruit}
    </li>
  )}
</For>
```

Output:

```html
<li>1. Apple</li>
<li>2. Banana</li>
<li>3. Cherry</li>
```

## Updating Lists

### Adding Items

```jsx
function AddItemExample() {
  const items = cell(['Item 1']);

  const addItem = () => {
    set(items, [...get(items), `Item ${get(items).length + 1}`]);
  };

  return (
    <div>
      <button onclick={addItem}>Add Item</button>
      <ul>
        <For each={items}>
          {(item) => <li>{item}</li>}
        </For>
      </ul>
    </div>
  );
}
```

### Removing Items

```jsx
function RemoveItemExample() {
  const items = cell(['A', 'B', 'C', 'D']);

  const removeItem = (index) => {
    set(items, get(items).filter((_, i) => i !== index));
  };

  return (
    <ul>
      <For each={items}>
        {(item, index) => (
          <li>
            {item}
            <button onclick={() => removeItem(index)}>×</button>
          </li>
        )}
      </For>
    </ul>
  );
}
```

### Reordering Items

```jsx
function ReorderExample() {
  const items = cell([1, 2, 3, 4, 5]);

  const shuffle = () => {
    const shuffled = [...get(items)].sort(() => Math.random() - 0.5);
    set(items, shuffled);
  };

  return (
    <div>
      <button onclick={shuffle}>Shuffle</button>
      <ul>
        <For each={items}>
          {(num) => <li>Item {num}</li>}
        </For>
      </ul>
    </div>
  );
}
```

## How Reconciliation Works

When you update a list, Rift uses the **Longest Increasing Subsequence (LIS)** algorithm to determine the minimum number of DOM operations needed:

```jsx
// Before: [A, B, C, D]
// After:  [D, A, B, C]

// Rift figures out that only D needs to move
// A, B, C stay in place
```

Items are tracked by **reference equality**. This means:

```jsx
// Objects maintain identity
const items = cell([
  { id: 1, name: 'Alice' },
  { id: 2, name: 'Bob' },
]);

// Updating preserves DOM nodes for unchanged items
set(items, [
  get(items)[0],                    // Same object = same DOM node
  { id: 2, name: 'Bobby' },         // New object = new DOM node
]);
```

<Aside type="tip">
For best performance with objects, maintain stable references for unchanged items instead of creating new objects every time.
</Aside>

## Nested Lists

You can nest `<For>` components:

```jsx
function NestedListExample() {
  const categories = cell([
    { name: 'Fruits', items: ['Apple', 'Banana'] },
    { name: 'Vegetables', items: ['Carrot', 'Broccoli'] },
  ]);

  return (
    <div>
      <For each={categories}>
        {(category) => (
          <div>
            <h3>{category.name}</h3>
            <ul>
              <For each={cell(category.items)}>
                {(item) => <li>{item}</li>}
              </For>
            </ul>
          </div>
        )}
      </For>
    </div>
  );
}
```

## Todo List Example

Here's a complete todo list demonstrating common patterns:

```jsx
import { defineComponent, cell, get, set } from 'rift-js';

function TodoApp() {
  const todos = cell([
    { id: 1, text: 'Learn Rift', done: true },
    { id: 2, text: 'Build something', done: false },
  ]);
  const newTodo = cell('');
  let nextId = 3;

  // Derived cells
  const remaining = cell(() =>
    get(todos).filter(t => !t.done).length
  );

  const addTodo = () => {
    const text = get(newTodo).trim();
    if (!text) return;

    set(todos, [
      ...get(todos),
      { id: nextId++, text, done: false }
    ]);
    set(newTodo, '');
  };

  const toggleTodo = (id) => {
    set(todos, get(todos).map(t =>
      t.id === id ? { ...t, done: !t.done } : t
    ));
  };

  const removeTodo = (id) => {
    set(todos, get(todos).filter(t => t.id !== id));
  };

  return (
    <div>
      <h1>Todos ({get(remaining)} remaining)</h1>

      <form onsubmit={(e) => { e.preventDefault(); addTodo(); }}>
        <input
          type="text"
          value={get(newTodo)}
          oninput={(e) => set(newTodo, e.target.value)}
          placeholder="What needs to be done?"
        />
        <button type="submit">Add</button>
      </form>

      <ul>
        <For each={todos}>
          {(todo) => (
            <li class={todo.done ? 'done' : ''}>
              <input
                type="checkbox"
                checked={todo.done}
                onchange={() => toggleTodo(todo.id)}
              />
              <span>{todo.text}</span>
              <button onclick={() => removeTodo(todo.id)}>×</button>
            </li>
          )}
        </For>
      </ul>
    </div>
  );
}

defineComponent('todo-app', TodoApp);
```

## Empty State

Handle empty lists by checking the array length:

```jsx
function ListWithEmptyState() {
  const items = cell([]);

  return (
    <div>
      {get(items).length === 0 ? (
        <p>No items yet. Add some!</p>
      ) : null}
      <ul>
        <For each={items}>
          {(item) => <li>{item}</li>}
        </For>
      </ul>
    </div>
  );
}
```

Or use `<Show>` for conditional rendering:

```jsx
function ListWithEmptyState() {
  const items = cell([]);
  const hasItems = cell(() => get(items).length > 0);

  return (
    <div>
      <Show when={hasItems} fallback={<p>No items yet!</p>}>
        <ul>
          <For each={items}>
            {(item) => <li>{item}</li>}
          </For>
        </ul>
      </Show>
    </div>
  );
}
```

## Performance Tips

### Prefer Immutable Updates

```jsx
// ✅ Create new array - Rift can track changes
set(items, [...get(items), newItem]);

// ❌ Mutating won't trigger updates
get(items).push(newItem);
notify(items); // Requires manual notify
```

### Stable Keys

While Rift tracks by reference, keeping stable object references helps:

```jsx
// ❌ Creates new objects every render
const filteredItems = items.filter(i => i.active);

// ✅ Filter once and maintain references
const filteredItems = cell(() => get(items).filter(i => i.active));
```

### Avoid Inline Objects in Handlers

```jsx
// ❌ Creates new function each render
<For each={items}>
  {(item) => (
    <button onclick={() => handleClick(item)}>
      {item.name}
    </button>
  )}
</For>

// ✅ This is actually fine in Rift due to event delegation
// The handler is set once when the item is created
```

## Compiled Output

Understanding the compilation helps with debugging:

```jsx
// Input
<For each={todos}>
  {(todo) => <li>{todo.text}</li>}
</For>

// Compiles to something like:
forBlock(container, todos, (anchor, todo, index) => {
  const li_1 = $tmpl_1().firstChild;
  const li_1_text = li_1.firstChild;
  li_1_text.nodeValue = todo.text;
  anchor.before(li_1);
  return { start: li_1, end: li_1 };
});
```
