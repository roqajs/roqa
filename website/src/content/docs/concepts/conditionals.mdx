The `<Show>` component conditionally renders content based on a boolean condition. It efficiently creates and destroys DOM nodes as the condition changes.

## Basic Usage

```jsx
import { defineComponent, cell, get, set } from 'rift-js';

function ToggleExample() {
  const visible = cell(true);

  return (
    <div>
      <button onclick={() => set(visible, !get(visible))}>
        Toggle
      </button>
      
      <Show when={visible}>
        <p>This content is visible!</p>
      </Show>
    </div>
  );
}

defineComponent('toggle-example', ToggleExample);
```

<Aside>
`<Show>` is a **compile-time** component like `<For>`. It's one of only two PascalCase JSX elements allowed in Rift.
</Aside>

## The `when` Prop

Pass a cell or expression to the `when` prop:

```jsx
// With a cell
const isLoggedIn = cell(false);

<Show when={isLoggedIn}>
  <p>Welcome back!</p>
</Show>

// With a derived expression
const items = cell([1, 2, 3]);

<Show when={cell(() => get(items).length > 0)}>
  <p>You have items!</p>
</Show>
```

## Fallback Content

Use the `fallback` prop to show content when the condition is false:

```jsx
function LoginStatus() {
  const user = cell(null);

  return (
    <Show 
      when={cell(() => get(user) !== null)}
      fallback={<p>Please log in</p>}
    >
      <p>Welcome, {get(user)?.name}!</p>
    </Show>
  );
}
```

## Common Patterns

### Loading States

```jsx
function DataLoader() {
  const loading = cell(true);
  const data = cell(null);
  const error = cell(null);

  this.connected(() => {
    fetch('/api/data')
      .then(res => res.json())
      .then(d => {
        set(data, d);
        set(loading, false);
      })
      .catch(e => {
        set(error, e.message);
        set(loading, false);
      });
  });

  return (
    <div>
      <Show when={loading}>
        <p>Loading...</p>
      </Show>
      
      <Show when={error} fallback={null}>
        <p class="error">Error: {get(error)}</p>
      </Show>
      
      <Show when={cell(() => !get(loading) && !get(error) && get(data))}>
        <div class="data">
          {/* Render data */}
        </div>
      </Show>
    </div>
  );
}
```

### Permissions

```jsx
function AdminPanel() {
  const user = cell({ role: 'user' });
  const isAdmin = cell(() => get(user).role === 'admin');

  return (
    <div>
      <h1>Dashboard</h1>
      
      {/* Regular content */}
      <p>Welcome to the dashboard</p>
      
      {/* Admin-only section */}
      <Show when={isAdmin}>
        <section class="admin-tools">
          <h2>Admin Tools</h2>
          <button>Manage Users</button>
          <button>View Logs</button>
        </section>
      </Show>
    </div>
  );
}
```

### Form Validation

```jsx
function SignupForm() {
  const email = cell('');
  const password = cell('');
  
  const emailError = cell(() => {
    const e = get(email);
    if (!e) return '';
    if (!e.includes('@')) return 'Invalid email address';
    return '';
  });
  
  const passwordError = cell(() => {
    const p = get(password);
    if (!p) return '';
    if (p.length < 8) return 'Password must be at least 8 characters';
    return '';
  });

  return (
    <form>
      <div>
        <input
          type="email"
          value={get(email)}
          oninput={(e) => set(email, e.target.value)}
          placeholder="Email"
        />
        <Show when={cell(() => get(emailError) !== '')}>
          <span class="error">{get(emailError)}</span>
        </Show>
      </div>
      
      <div>
        <input
          type="password"
          value={get(password)}
          oninput={(e) => set(password, e.target.value)}
          placeholder="Password"
        />
        <Show when={cell(() => get(passwordError) !== '')}>
          <span class="error">{get(passwordError)}</span>
        </Show>
      </div>
    </form>
  );
}
```

### Expandable Sections

```jsx
function Accordion() {
  const sections = cell([
    { title: 'Section 1', content: 'Content 1', open: false },
    { title: 'Section 2', content: 'Content 2', open: false },
    { title: 'Section 3', content: 'Content 3', open: false },
  ]);

  const toggleSection = (index) => {
    set(sections, get(sections).map((s, i) => 
      i === index ? { ...s, open: !s.open } : s
    ));
  };

  return (
    <div class="accordion">
      <For each={sections}>
        {(section, index) => (
          <div class="section">
            <button onclick={() => toggleSection(index)}>
              {section.title}
            </button>
            <Show when={cell(() => get(sections)[index].open)}>
              <div class="content">
                {section.content}
              </div>
            </Show>
          </div>
        )}
      </For>
    </div>
  );
}
```

## Nested Conditionals

You can nest `<Show>` components:

```jsx
function NestedConditions() {
  const step = cell(1);

  return (
    <div>
      <Show when={cell(() => get(step) >= 1)}>
        <div>Step 1: Basic Info</div>
        
        <Show when={cell(() => get(step) >= 2)}>
          <div>Step 2: Contact Details</div>
          
          <Show when={cell(() => get(step) >= 3)}>
            <div>Step 3: Review & Submit</div>
          </Show>
        </Show>
      </Show>
      
      <button onclick={() => set(step, get(step) + 1)}>
        Next Step
      </button>
    </div>
  );
}
```

## Show vs Ternary

Both approaches work in Rift:

### Using `<Show>`

```jsx
<Show when={isVisible} fallback={<p>Hidden</p>}>
  <p>Visible</p>
</Show>
```

Best for:
- Complex content blocks
- Content with its own state/lifecycle
- Multiple nested conditions

### Using Ternary

```jsx
{get(isVisible) ? <p>Visible</p> : <p>Hidden</p>}
```

Best for:
- Simple text changes
- Single element swaps
- Inline conditional classes

### Using Logical AND

```jsx
{get(isVisible) && <p>Visible</p>}
```

Best for:
- Show/hide without fallback
- Simple conditional rendering

## How It Works

`<Show>` compiles to a `show_block()` runtime call:

```jsx
// Input
<Show when={visible}>
  <p>Content</p>
</Show>

// Compiles to something like:
show_block(container, visible, (anchor) => {
  const p_1 = $tmpl_1().firstChild;
  anchor.before(p_1);
  return { start: p_1, end: p_1 };
});
```

When the condition changes:
- **True → False**: Content is removed from DOM
- **False → True**: Content is created and inserted

This is different from CSS `display: none`—the DOM nodes are actually created and destroyed.

<Aside type="tip">
If you need to preserve state across hide/show cycles, keep the state in a cell outside the `<Show>` block, or use CSS visibility instead.
</Aside>

## Performance Considerations

### Heavy Content

For content that's expensive to create, consider whether `<Show>` or CSS hiding is better:

```jsx
// DOM is created/destroyed each toggle
<Show when={visible}>
  <heavy-chart-component></heavy-chart-component>
</Show>

// DOM stays in place, just hidden
<div style={get(visible) ? '' : 'display: none'}>
  <heavy-chart-component></heavy-chart-component>
</div>
```

### Derived Conditions

Create derived cells for complex conditions instead of inline expressions:

```jsx
// ✅ Good - single derived cell
const showAdvanced = cell(() => 
  get(user).role === 'admin' && get(settings).advancedMode
);

<Show when={showAdvanced}>
  <advanced-panel></advanced-panel>
</Show>

// ❌ Less clear - inline complex logic
<Show when={cell(() => get(user).role === 'admin' && get(settings).advancedMode)}>
  <advanced-panel></advanced-panel>
</Show>
```
